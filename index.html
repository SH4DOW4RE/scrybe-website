Encrypted before it leaves your browser

Private bookmark and history sync across browsers.

Scrybe is a browser extension and hosted sync service for privately saving and synchronizing bookmarks and browsing history across Chrome, Firefox, and compatible browsers. Its purpose is to let a user keep the same encrypted bookmark and history library across their own browsers, while preventing the hosted backend from reading the synced content.

Application purpose

What Scrybe does.

Scrybe provides encrypted bookmark and browsing history sync for browser extension users. The extension saves page URLs, titles, favicons, and visit timestamps locally, encrypts synced records in the browser, and sends only encrypted sync data to the configured backend.

For users

Use Scrybe to access your saved bookmarks and recent browsing history from your own paired browsers without creating an email/password account.

For sync

The hosted backend stores encrypted records and device metadata needed to synchronize browsers. It is not designed to decrypt bookmark or history content.

For operators

Advanced users can configure the extension to use a Scrybe-compatible backend they operate, because the backend URL is editable in extension settings.

Built for private, practical sync.

Scrybe focuses on the workflows people expect from browser sync, while keeping the security model simple: clients encrypt and decrypt, the backend stores opaque encrypted records.

Bookmarks with real metadata

Save the current page with its URL, title, and favicon. Long titles stay readable in the popup without losing the complete encrypted record.

Automatic history capture

Visited pages are saved automatically. Clicking a history entry opens it in a new tab, matching normal browser behavior.

Cross-browser accounts

Create an anonymous account once, then pair Chrome, Firefox, and compatible browsers using short-lived pairing codes.

Recovery code included

Scrybe generates a recovery code when an account is created, so users can recover access if every browser disconnects.

Device control

See paired browsers in settings and disconnect any browser that should no longer be allowed to sync.

Backend choice

The extension settings include the backend URL. Use the hosted Shadoweb API or point Scrybe at infrastructure you control.

Self-hostable by design

Use Scrybe with your own backend.

The official hosted API is the easiest path for most users, but Scrybe does not lock sync to a single server. If you operate your own backend, change the backend URL in the extension settings and keep using the same client-side encryption model.

Change the sync endpoint

In the extension popup, open Settings and replace the backend URL with your own Scrybe-compatible API endpoint.

How Scrybe connects browsers.

No username, email, or password is required. Pairing is designed for browser-to-browser setup, while recovery codes are for restoring an account when no paired browser is available.

Create a new account

The extension creates an anonymous account, generates local encryption keys, and shows a recovery code to save.

Pair another browser

Use a temporary pairing code from an already connected browser to sync another browser into the same account.

Recover when needed

If the extension disconnects everywhere, enter the saved recovery code to reconnect a browser.

Private sync without server-side secrets.
